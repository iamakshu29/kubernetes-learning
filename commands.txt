Taint and Tolerance
kubectl get nodes - to get all the nodes (master + worker)

to Taint command is
kubectl taint nodes <node-name> key=value:NoSchedule
>> kubectl taint node worker-node-1 prod=true:NoSchedule
--- key=value helps to identify the taint easily when you have multiple taints on a node.

to Remove Taint command is
kubectl taint nodes <node-name> key=value:NoSchedule-
>> kubectl taint node worker-node-1 prod=true:NoSchedule-

If all the worker-nodes are tainted and we create or apply the pod, then it will be in pending state.

Tolerance - Run the pods even on Tainted nodes
add this in spec: 

tolerations:
  - key: "prod"
    operator: "Equal"
    value: "true"
    effect: "NoSchedule"

Init or Sidecar Container
to check a particular container as when we use Init or Sidecar container, logs command shows logs of one container only. (default one can be init or main)
To get the logs of specific container either init, main or sidecar. Command :
kubectl logs <podname> -c <containername> -n <namespace>
>> kubectl logs init-test -c init-cont -n default
